1 Introducton

Phishing is one of major personal security risks in Internet this days. Most websites, mostly the popular ones and which are in use by non-tech savvies are a great findings for phishing techniques. This websites might be, but not limited to, social networks, banks websites, email gateways and so on. In this document only websites phishing techniques will be reviewed, I won't touch social phishing, email scamming and such at all.

1.1 Statistic facts 1

All data provided here taken from AntiPhishing Work Group

  • Number of phishing websites averages to 121,000 per month in Q3 2016.
  • Number of brands targeted by phishing compaigns averages to 350 per month
  • Total number of phishing websites attacks was 364,424 in Q3 2016.
  • Highest number of phishing websites attacks was in Q2 2016 with 466,065 attacks. This also was the all-time-highest number.
  • Most targeted industry sectors are as follows by descending order:
    1. Retail/Service :: 43%
    2. Financial :: 21%
    3. ISP :: 12%
    4. Payment Services :: 10%
    5. Unclassified :: 5%
    6. Multimedia :: 4%
    7. Social Networking :: 2%
    8. Government :: 1%
  • Countries that are most infected countries:
    1. China
    2. Taiwan
    3. Turkey
  • Less infected countries:
    1. Norway
    2. Finland
    3. Sweden
  • American banks estimate that at least $300 is lost for every hour a phishing website is up and running.

2 Phising targets

As it can be seen from statistics breakdown above the targets for phishing are mostly Service, Retail and Financial sectors in countries where IT culture levels are not high. The victims are mostly non-tech people who blindly click on everything that appear on their screen allowing by this installation of malware and providing willingly their credentials and personal info to non-legit third parties.

3 Why Phishing works

Phishing attacks are more than just techniques. Studies show2 that this attacks are more social engeneering attacks and according to more studies 2, 3, 4 more than 90% of people depend on website's design to verify it's authencity. Well designed phishing websites that look completely like the originals or implement most valuable parts of original design are more like to fool person who comes there. There is another problem found in how anti phishing applications work. If the application is too intrusive, shows too many pop-ups it more likely to be silenced and, eventually, disabled by the user5, 6. Anti phishing applications that show unobtrusive warnings are more likely to be ignored by the end user. All of this makes phishing prevention very hard on human factor level.

4 Phishing techniques 7

Most common phishing types for web browsers are:

  1. Link (url?)Manipulation :: This technique uses disques to hide malicius website behind an URL of well known websites. This might be misspelled URLs which look very similar to the original but lack or have different just one letter, URLs with wrong TLD, correct website name though in practice are subdomains of different websites. With introduction of IDN another level of danger were added where phishing domain can look exactly as original one by substituting one letter with letter from different language alphabet. Some examples are:
    http://fасеbook.com
    This one seems completely legit but in fact half of letters here are in Russian.
    http://goog1e.com
    This one is a bit easier to spot but it depends on the font installed for the web browser.
    http://gooogle.com or http://faceboook.com
    Simple mistypes, nevertheless most people fail to notice the difference at first.
  2. Website Forgery :: Usually this one is coupled with link manipulation technique but it's not a must. For this type of attack scammer designing a website which looks completely same as target. When victim enters this phishing website he can't say the difference including the URL which changed by JavaScript to masquerade the website completely. Later on the scammer might request from victim to login to what seems legit website and provide an oAuth login pop-up for his google (Facebook or any other oAuth provider) website. After victim accepts it - scammer gets the authtoken for this user and depending on privileges that he was able to approve for his website. This technique might also be used to get login credentials for websites which are not using oAuth log in authentication.
  3. Covert Redirect :: This technique uses links that look like legit on original websites but instead are carefully forged redirect links that take victim to an attacker's web site. The rest from here is just like in website forgery, after victim clicks on the link he might receive, for example, oAuth login pop-up for which if he agrees he providing access to his personal information. The main problem with this type of attack is that it is very hard to spot as attacker might use original website for this attack. It is possible, under certain circumstances, to inject required links to websites like Facebook which will look completely legitimate. (tests on facebook)

5 Available Solutions and their problems

Solutions might be separated in 3 categories, each one with it pros and cons. Most anti phishing software rely on combination between this approaches to provide best hit rate.

5.1 Whitelists

This solution provides a list of whitelisted URLs which are safe to access and proven as non-phishing websites. As much as this solution is good it also have some flaws:

  • This is virtually impossible to include all of legitimate websites on internet into a whitelists.
  • User still have to verify manually that the website is legit as it is possible, under some curcumstances, to fool whitelist checker and feed it with good URL while taking victim to phishing website

5.2 Blacklists

This solution provides a lits of phishing sites and preventing end user to open the targeted websites. The problem with this solution mainly lies in fact that according to APWG research most of the phishing websites are short living. An avarage life time span for them is less than 40 hours for 20168. Most of such websites, and I'm talking about avarage of 4033 attacks per day1, 8, are not reported to corresponding blacklists of antiphishing software therefore are not blocked for the end user.

5.3 Heuristic aproaches

In this category are solution based on different researches which try to identify phishing resource on-fly. This approaches might use different types of data collected from the page to set a trust score and act according to the score received. They might take in consideration URLs, links on page, domain age, spelling errors, known images/files from high risk websites, etc. For example SpoofGuard9 checks the current domain name, then the full URL is analyzed to de-tect obfuscation as well as non-standard port numbers, then Spoofguard analyzes the contents, making note of any password felds, embedded links, and images. The problem with such approaches is:

  • It can be fooled if alghorithm of the approach is known.
  • It can hit false positive on legitimate websites
  • It can miss phishing websites.

6 Proposed solution

In the world of open software we have more power to secure the web and build trust network. The proposed solution consists of combination of all 3 approaches but extended to limitless extend of community contribution.

6.1 Phishing detection

Detecting phishing websites are done by combination of Whitelists, blacklists and heuristic approaches where the heuristic approach is extendable by community. The checkers are sitting on remote server and are voted by community. In a long run I belive that the most valuable checkers will be promoted to be first. While this approach obviously have a drawback of lag for end user it could be used in an async way where website is not loading until WL and BL are checked and then the website is shown with disabled forms/buttons to prevent user from clicking while the heuristic checkers are running.

6.2 User notifications

As it was discussed earlier, users don't like intrusive notifications and tend to disable or ignore them if such happen. To address this issue I propose the following:

  1. Website is not loading until the Whitelist and Blacklist checks finish
  2. Website loads with all links, buttons and forms set to "disabled" to prevent user from clicking on them until the heuristic checks are done.
  3. If website is clearly a phishing website the content is overlayed and blocked with explanation and a link to report non-phishing website to review if user thinks it's a mistake
  4. If website might be phishing but it's not clear it's design is modified in a very clear way to make user pay attention to it. At the top of the website should be a bar shown with explanation why the websites look is modified and suggestions on what and how to check with link to report non phishing website as well.

6.3 Detection updates

  1. Whitelists and blacklists are hashes of URLs stored on servers.
  2. Checks are stored on Github page and allow contribution of community.
  3. Extension is distributed through ChromStore and autoupdated as required.
